Cloud Computing - kort innføring

  • Handler om IT-drift; dukket opp rundt 2006, tok av rundt 2009.
  • Står i sterk kontrast til trad. drift som krever store investeringer for å komme igang/skalere
  • Kort oppsumert: Datakraft blir en ressurs på linje med strøm. Bruk det du trenger, betal for det du bruker
  • Flere modeller; viktigste er IaaS og PaaS (Infrastructure/Platform as a Service)
  • Hva er AWS?
    • 30+ ulike tjenester innenfor virtualisering, database, datalagring/content delivery, applikasjonstjenester++
    • Hovedfokus er virtualisering (EC2) og datalagring (S3)
    • Tilgjengelig gjennom dashboard, webservices og developer tools (kommandolinje)
    • Regioner og soner
  • Husk:
    • Elastic Beanstalk
    • AWS Free Tier - Gratis tilgang
      i et år

Elastic Compute Cloud (EC2)

  • Virtualiseringstjeneste; blant de første tjenestene i AWS
  • Baserer seg på Amazon Machine Images (AMIs)
    • Offisielle AMIs
    • Community / Marketplace
  • 15 ulike maskintyper (i EU)
  • Kan skaleres etter behov (AutoScale)
  • Kan kobles med Elastic Load Balancer for lastbalansering

EC2 - Prising

  • Betales per påbegynte time
  • Prismodeller: on demand, reserved og spot

Lagringstjenester

  • Elastic Block Storage (EBS), Simple Storage Service (S3)
  • EBS mountes direkte på EC2; maks 2 TB per volum. Kan klones og snapshottes
  • S3 brukes typisk for statiske filer; uendelig kapasitet
  • Glacier

Databasetjenester

  • Amazon Relational Database Service (RDS, SQL), DynamoDB (NoSQL) og Redshift (Datavarehus)
  • RDS er instansbasert (tilsvarende EC2); DynamoDB er en ren service; Redshift er clusterbasert
  • RDS har maksbegrensning på 3TB, DynamoDB 5PB+, Redshift max 1,6PB per cluster

Erfaringer og utfordringer

  • Feil vil oppstår; soner og regioner vil gå ned
  • God planlegging og risikovurdering er et must
  • Applikasjoner må tilpasses
  • Økonomi
